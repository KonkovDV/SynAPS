# Support

## What This Repository Supports Today

This repository supports public discussion and contribution around the current SynAPS engineering baseline:

1. reproducible solver or schema bugs;
2. benchmark harness issues or gaps in reproducibility;
3. packaging, install, CI, and release-surface problems;
4. bounded documentation and publication-surface corrections;
5. domain parametrization proposals grounded in real constraints or datasets.

## What Is Not A Support Channel

1. private vulnerability reports in public issues;
2. emergency production support;
3. confidential plant data, customer schedules, or regulated records;
4. open-ended consulting requests without a reproducible engineering task.

## Read First

Before opening a request, check:

1. `README.md`
2. `benchmark/README.md`
3. `docs/README.md`
4. `CONTRIBUTING.md`
5. `SECURITY.md` for vulnerability reporting

## Recommended Public Channels

1. reproducible bugs or regressions: GitHub Issues with the provided templates;
2. proposed fixes: Pull Requests with validation notes;
3. domain or roadmap ideas: GitHub Issues using the feature or domain templates.

## Response Expectations

SynAPS is a research and engineering repository, not a staffed product support desk.

Best-effort triage is the intended operating model until a dedicated maintainer team is explicitly declared.
