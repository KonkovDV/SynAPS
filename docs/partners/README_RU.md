## Что изменилось

Раньше в активной папке было слишком много параллельных версий, специальных презентаций и файлов с методическими пояснениями.

Теперь активный слой короче специально:

- маршрут по умолчанию теперь начинается с проверки утверждений → техническая верификация → доказательства бенчмарков
- один путеводитель вместо путеводителя и отдельного манифеста
- обзор покрытия верификации теперь идёт в маршруте технической проверки по умолчанию, а глубокий аудит остаётся дополнительным хранилищем доказательств; исторический снимок вынесен в архив
- в активном слое остаётся короткая русская сводная версия, а полный русский партнёрский документ сохранён в архивном маршруте
- меньше презентационных файлов в основном маршруте

Старые и дублирующие материалы перенесены в [../../archive/partner-2026-04-slimming/README.md](../../archive/partner-2026-04-slimming/README.md).

## Текущая позиция

Сегодня сильная сторона SynAPS — открытое и проверяемое детерминированное ядро планирования с 13 конфигурациями решателей, текущей собираемой выборкой из 175 тестов, ограниченным ремонтом, воспроизводимой верификацией, минимальной сетевой границей на TypeScript и задокументированными вариантами LBBD-HD в портфеле решателей.

Слабая сторона — отсутствие подтверждённых пилотов, зрелой вертикальной интеграции и эксплуатационной истории уровня промышленного APS.

Активный партнёрский пакет теперь отражает именно эту границу, без лишнего шума.
